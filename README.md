# ioguard

**Deterministic safety scanning for the LLM I/O boundary.** `ioguard` inspects text
crossing into and out of language models for credential leaks and Unicode/prompt-injection
smuggling — fast, offline, no ML, no network. It ships as a CLI (`ioguard scan`), a Rust
library (`ioguard-core`), and a C-ABI (`ioguard-ffi`) so the same detection logic guards a
subprocess gate and a hot-path daemon from one source of truth.

## Why

Untrusted text reaching an LLM (user input) or returned by one (model output) is an attack
surface: secrets get pasted into prompts and logged; invisible Unicode and control tokens
smuggle instructions past human review. `ioguard` is the deterministic gate that runs on
every I/O pair in sub-milliseconds.

## Detectors (v0.1.0)

| Category | Catches |
|---|---|
| `secret` | API keys (Anthropic, OpenAI, AWS, GitHub, Google, Stripe, …), PEM/GPG private keys, Luhn-valid card PANs |
| `unicode_tags` | U+E0000–E007F Tag-block instruction smuggling (with a validated RGI subdivision-flag carve-out) |
| `zero_width` | Invisible / `Default_Ignorable` format chars used to fragment secrets or instructions |
| `bidi` | Trojan-Source bidirectional controls; locale-aware via structural BCP-47 |
| `homoglyph` | UTS#39 mixed-script + NFKC confusable spoofs (Cyrillic/Greek/fullwidth/math) |
| `special_token` | Chat-template / control tokens (ChatML, Llama3, Gemini, Mistral, Cohere, Falcon, …) |

## Install / build

```sh
cargo build --release          # ioguard CLI + libioguard_ffi.{a,dylib,so}
```

## CLI usage

`ioguard scan` reads stdin, writes Contract-A JSON to stdout, and sets an exit code:
`0` allow · `10` warn · `20` block.

```sh
printf 'sk-ant-api03-…' | ioguard scan          # → verdict: block (exit 20)
printf 'hello world'    | ioguard scan          # → verdict: allow (exit 0)
ioguard scan --direction output --locale ar < msg.txt
```

## Contract-A (the JSON schema)

Stable output schema:

```json
{
  "verdict": "allow | warn | block",
  "findings": [
    {
      "rule_id": "secret.anthropic_key",
      "category": "secret",
      "severity": "block | warn",
      "direction": "input | output | both",
      "span": [0, 108],
      "preview": "sk-ant-a..."
    }
  ],
  "stats": { "bytes_scanned": 108, "findings_count": 1, "duration_us": 10413 }
}
```

`preview` is always truncated — the full secret is never emitted. `verdict` is the max
severity across findings (`block` > `warn` > `allow`).

## Library / C-ABI

Native consumers link `ioguard-core::scan(text, &opts) -> ScanResult`. Cross-language
consumers use the panic-isolated C-ABI (`ioguard-ffi`):

```c
char *ioguard_scan(const uint8_t *text, size_t len, const char *opts_json); // owned Contract-A JSON
void  ioguard_free(char *json);                                              // free it; NULL-safe
```

Header generated by cbindgen at `include/ioguard.h`. The caller owns the returned string.

## Robustness

Every detector is backed by a Unicode/structural **property** (Default_Ignorable,
Extended_Pictographic, UTS#39 skeleton, structural BCP-47) rather than a fixed codepoint
list where possible, and is gated by a conformance corpus (`corpus/must-block`,
`corpus/must-allow`) that CI runs on every build. v0.1.0 was hardened across six adversarial
red-team rounds (the last two fully autonomous) to **zero in-scope high-severity findings**;
remaining lower-severity coverage gaps (sibling credential prefixes, additional
invisible-character families) are tracked for v1.1.
